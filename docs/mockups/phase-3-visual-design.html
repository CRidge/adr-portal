ADR-024: Introduce API Gateway Response Caching

Goal: reduce median read latency for public catalog endpoints.

Proposed

Domain

Platform Architecture

Decision Date

2026-03-12

Owner

Platform Team

Reviewers

Security, SRE, API Guild

Decision Summary

Use route-level TTL caching in the API gateway for read-heavy endpoints, with explicit bypass for authenticated and mutation requests.

Options Considered

  • Service-level in-memory caching
  • Dedicated distributed cache per bounded context
  • Gateway-managed response caching (recommended)

Review Timeline

  • 2026-03-07 Draft submitted by Platform Team.
  • 2026-03-09 Security review requested for cache poisoning safeguards.
  • 2026-03-12 Architecture board decision meeting scheduled.
